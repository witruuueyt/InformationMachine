Game4Automation Conveyor Library

This library is an extension for Game4Automation Starter or Game4Automation Professional. This library can be obtained from the Unity Asset Store or directly from us. To use the Conveyor Library, it is mandatory to install Game4Automation Starter or Game4Automation Professional first. Please always use the latest versions of Game4Automation Starter or Professional with the latest Conveyor Library.

The Game4Automation Conveyor Library is a library of parametric conveyor components. The included library elements can be used to build simulation models of intra-logistics and material flow systems. The transport of goods is physically simulated, i.e. the goods move freely on the conveyor system as in reality and can be transported and moved via physical forces.

All included standard components use functions of Game4Automation Starter or Professional. In particular, the elements of the Conveyor Library are equipped with drives and sensors. These can be controlled directly via script for simulation. For virtual commissioning, the simulation model can be connected to industrial controllers using the automation interfaces of the Starter or Professional version.

Installation and first steps

Please install the Game4Automation Starter or Professional version first via the Unity Asset Store or via Assets > Import Package > Custom Package. Afterwards you can install the Conveyor Library via the same procedure.

It is best to start viewing the demo scene of the sorter system directly after the installation. You can find the demo scene under Assets/game4automation-ConveyorLibrary/DemoScenes/SorterSystem.unity.

In addition to the sorter system demo scene, you will also find several smaller demos in the same folder.

Using the Game4Automation menu gam4automation > Conveyor Library Window an additional window can be opened, that provides special functions for the library:

This window can be used to control all conveyors or to set their default speed. It is also possible to speed up all drives via speed override, while keeping the simulation time. I.e. the drives become physically faster. It is also possible to change the physics timescale, so that the simulation runs faster or slower like in a time lapse.

Via Add Library Objects the library elements can be placed in the scene. Alternatively, the library objects (these are Unity prefabs) can be placed in the scene simply by dragging and dropping them from the project folder.

Snapping Objects

Some of the library components have snap points. The snap points can be used to place the objects together at the appropriate points.

Please note that for the function of the snap points, the gizmos must be activated. To do this, click on Gizmos in the upper right corner of the scene.

General object structure

All objects have a common structure. As soon as the parameters are changed in the inspector, the object is built automatically. Additional prefabs may be created automatically as subobjects for this purpose. In the standard, some subobjects are hidden in order to obtain a clearer representation. Optionally Is Editable can be switched on in all library elements in the inspector. All child elements will then be displayed.

For Sensor, Drives, and Transport Surfaces the Game4Automation standards are used. More information can be found in the general Game4Automation documentation.

Library Components

You can find all included Library components in the folder Assets/game4automation-ConveyorLibrary.

Straight Conveyor

The Straight Conveyor can be displayed as a roller conveyor or belt conveyor. It has SnapPoints on each end and on the sides. If another conveyor is snapped to the side the Side Guide will be removed. Each conveyor has automatically a sensor at the end.

Curved Conveyor

The Curved Conveyor can be displayed as a roller conveyor or belt conveyor. It has SnapPoints on each end. Each conveyor has automatically a sensor at the end.

Straight Conveyor Group

The Straight Conveyor Group automatically creates multiple straight conveyors as sub components.

Aligner

The aligner is able to align the MUs (movable unity) with the pivot point of the Aligner. Additionally it is possible to add rotational or linear forces to the MU.

Chain Transfer

The chain transfer is a chain which can lift MUs (Movable Units) and transport them to the side. The chain transfer has two drives included. One for lifting and one for transporting the goods.

Guide Rolls

The Guide Rolls can be places onto a conveyor. They are pushing the goods into a certain direction with a certain speed. Two drives are included. One for turning the rolls and another for selecting the direction.

Sensor

The sensor is based on the standard Game4Automation Sensor. Additionaly it has some parameters specially for conveyor systems and can be places separately or as a sub component on conveyors. If a sensor is placed as a subcomponent on conveyors it is automatically aligned with the Conveyor direction and position.

Source

The source is based on the standard Game4Automation Source. Goods with different dimensions and weights can be automatically created. The source is always trying to keep a defined distance between the produced goods. If you want to have a different look of the goods you can create your own source and exchange the included meshes.

Sink

The sink is destroying goods which are intersecting with the Sink area, which is a box collider.

SourceTote

The SourceTote is a second source, just with a different 3d mesh than the standard source.
